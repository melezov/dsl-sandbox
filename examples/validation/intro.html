Validation can be included in entity or value objects to enforce additional integrity constraints.

Validation is defined using a validation block with one or more specifications. Every specification must specify two expressions:

  1. expression defining the validation constraint (can be expression or outside specification referenced by name)
  2. expression holding a message returned if validation has failed

Keyword check inside validation block enforces performing validation when persisting object. If the validation fails, persist operation fails and an exception is thrown. If check is omitted, validation is not executed on persisting. @todo validation can be executed explicitly or for some other use case
