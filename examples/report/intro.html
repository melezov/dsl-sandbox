Report should be used as container for various data. Data for complex forms can be populated at once instead of making multiple round trips to the server. It is also used for sending this data to the reporting engine.

Report has two types of properties:

  1. Simple properties are defined by type and used as report parameters
  2. Data properties are defined with type and an expression which behaves as specification and is used to filter data

Report is used by first filling out the simple properties, and then data properties can be populated by expressions specified in report.

In this example, CustomerOrders report is used to fetch orders by customer's ssn. ssn and totalOrder properties are used as simple properties. customer and orders are data properties.
